浅谈hugo搭建博客

花了足足三个小时，来搞这个博客，累的要死，终于算是大致搞明白了。还有很多不太明白的地方，就写一篇博客和大家分享

初试Hugo

Hugo是由Go语言实现的静态网站生成器。简单、易用、高效、易扩展、快速部署。

安装Hugo

  1. 前往官网下载Hugo。
  2. 解压Hugo，并将Hugo的解压目录引入环境变量的Path中。
  3. 通过hugo version命令测试是否引入成功。

建立一个站点，并添加默认主题

  1. 自定义一个空目录执行 hugo new site XXXXXXXX，X的位置可以用任意字符串替代。如果是要搭建在Github Page上。给出推荐的格式：<你的github名>.github.io-creator。
  2. 进入生成的目录 cd <你的github名>.github.io-creator。
  3. 将目录初始化为一个Git本地仓库git init。
  4. 下载默认的主题git submodule add https://github.com/budparr/gohugo-theme-ananke.git themes/ananke。
  5. 将选择默认主题追加到配置文件echo 'theme = "ananke"' >> config.toml。

添加一些内容

通过hugo new posts/my-first-post.md生成你自己的第一篇文章。不推荐这样做，测试后发现会直接在主目录下新生成posts文件夹，而不是生成在content目录下。

启动Hugo服务器

启动服务器：hugo server。通过本地http://localhost:1313/查看本地站点。 参数 -D可以显示草稿文章

构建静态页面

在根目录通过huge生成public文件夹。然后创建.gitignore文件写入/public/，用于忽略public文件夹。

建立Public本地仓库

  1. 在public文件夹内通过git init建立本地仓库。
  2. 添加并提交本地仓库。

对接Github

在github上创建新的云端仓库，仓库名为 <你的github名>.github.io，然后将本地仓库同步到云端仓库。

部署到Github Page
